# Code of Conduct

## Our Commitment

In the interest of fostering an open, inclusive, and professional environment, we as contributors, maintainers, and community members pledge to make participation in this project a respectful and harassment-free experience for everyone regardless of age, body size, disability, ethnicity, gender identity, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

This project promotes ethical cybersecurity research, responsible disclosure, defensive security practices, and collaborative learning.

---

# Our Standards

Examples of behavior that contribute to a positive environment include:

- Demonstrating professionalism and empathy
- Respecting differing viewpoints and experiences
- Providing constructive feedback
- Accepting responsibility for mistakes
- Using welcoming and inclusive language
- Supporting new contributors and learners
- Encouraging ethical and defensive cybersecurity practices
- Reporting vulnerabilities responsibly

Examples of unacceptable behavior include:

- Harassment, intimidation, or discrimination
- Trolling, insulting, or derogatory comments
- Publishing private information without permission
- Personal or political attacks
- Spamming or disruptive behavior
- Malicious cybersecurity activities
- Sharing malware, phishing kits, exploits, or illegal tools
- Encouraging unauthorized access or illegal hacking
- Posting harmful, abusive, or offensive content

---

# Ethical Cybersecurity Policy

This repository is intended strictly for:

- Educational purposes
- Security research
- Defensive cybersecurity operations
- SOC analysis workflows
- Threat intelligence learning

Contributors and users must NOT use this project for:

- Illegal hacking
- Unauthorized access
- Malicious attacks
- Credential theft
- Malware deployment
- Phishing campaigns
- Data theft
- Any unlawful activity

Any malicious usage discussion or contribution may result in immediate removal and reporting where appropriate.

---

# Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing standards of acceptable behavior and may take appropriate action in response to unacceptable behavior.

Maintainers reserve the right to remove, edit, or reject:

- Comments
- Commits
- Code
- Wiki edits
- Issues
- Pull requests
- Discussions

that violate this Code of Conduct.

---

# Enforcement Guidelines

## 1. Correction

Community impact: Minor inappropriate behavior.

Consequence:
- Private warning
- Request for correction

---

## 2. Warning

Community impact: Repeated inappropriate behavior.

Consequence:
- Formal warning
- Temporary interaction restrictions

---

## 3. Temporary Ban

Community impact: Serious violations.

Consequence:
- Temporary suspension from project participation

---

## 4. Permanent Ban

Community impact: Severe misconduct or malicious activity.

Consequence:
- Permanent removal from the community and repository spaces

---

# Reporting Guidelines

If you experience or witness unacceptable behavior, report it to the project maintainers.

Provide:
- Description of incident
- Relevant screenshots/logs if available
- GitHub username or related details

All reports will be reviewed confidentially and fairly.

---

# Scope

This Code of Conduct applies within all project spaces, including:

- GitHub Issues
- Pull Requests
- Discussions
- Wiki pages
- Documentation
- Social/community interactions related to the project

It also applies when an individual officially represents the project in public spaces.

---

# Security Disclosure

Please do NOT publicly disclose critical vulnerabilities.

Responsible disclosure is encouraged through private communication with maintainers before public release.
